WSL CANT FIND SCREEN FOR X410 OR OTHER XSERVER - SWEDEN - 2026

FIX errors for Xservers like x410 or GWSL in LINUX when you start Desktop via Windows 11 / WSL for LINUX.

FIRST THING TO DO

Open POWERSHELL from Microsoft and enter: ipconfig

Look after IP for wsl and you will see: Ethernet adapter vEthernet (WSL (Hyper-V firewall)):
Copy ip for wsl - something like this: IPv4 Address 182.18.122.9  <-- DEMO

Copy that IP nummber and look down in this text file - copy IP there.

After copy you can close POWERSHELL and Go back to ex debian/fedora/redhat/centos/ubuntu or what linux version you use.

-------------------------------------------------

Now Copy and paste in Linux Terminal

NOTE if you see ann error you need to install systemd-resolved

systemd-resolved are some for Redhat, Fedora, Ubuntu, Debian, Centos and so one.. Use google if you cant find it =)

ex: sudo apt install systemd-resolved or sudo dnf install systemd-resolved

-------------------------------------------------

Copy and paste in terminal for LINUX. No stress.

sudo systemctl unmask systemd-resolved.service
sudo systemctl enable systemd-resolved.service

sudo ls -l /etc/resolv.conf
sudo mv /etc/resolv.conf /tmp/resolv.conf.bak
sudo nano /etc/resolv.conf

nameserver 182.18.122.9 <-- put your ip here from powershell from microsoft

sudo chattr +i /etc/resolv.conf
sudo systemctl restart systemd-resolved
cat /etc/resolv.conf

Last line will lock file so windows and WSL2 not can change ip for WSL - Big problems fixed for WSL2 =]
